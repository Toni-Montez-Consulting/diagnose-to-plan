# Purpose

Turn a scoped plan into client-facing language that sells the working system, not tool novelty.

# Inputs Required

- Diagnose memo
- Plan roadmap
- offering type
- COI verdict

# Steps

1. Lead with owner value.
2. Name deliverables as a Client Operating Kit when appropriate.
3. Include exclusions and approval boundaries.
4. Include handoff, runbook, and eval requirements.

# Human Approval Trigger

Do not send or price final work without operator review.
