# Microsoft Kill Switch

Trigger this process if:

- outside-work approval changes
- Toni's role changes and conflict risk increases
- a client acquisition introduces Microsoft overlap
- Microsoft customer overlap surfaces
- approval is rescinded
- a client asks for Microsoft-influencing work

Immediate actions:

1. Pause affected engagements.
2. Remove or edit public claims that create conflict risk.
3. Notify affected clients with a short operational note.
4. Stop billing relationships that cannot continue cleanly.
5. Archive affected repo/context files locally.
6. Record the decision in the engagement decision log.
