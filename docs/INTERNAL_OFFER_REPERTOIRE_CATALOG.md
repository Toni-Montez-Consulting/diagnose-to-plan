case-study claim until it passes the relevant proof, redaction, permission,

## Catalog Rules

- Capture the useful pattern while it is fresh.
- Name the buyer problem before naming the internal tool.
- Separate internal repeatability from public proof.
- Promote only after real delivery shows the pattern is useful.
- Keep brand, legal, finance, calendar, and admin setup as practical service
  components before turning them into public standalone offers.
