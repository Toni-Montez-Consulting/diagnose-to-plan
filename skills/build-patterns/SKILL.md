# Build Patterns

The synthesis docs in `extracts/synthesis/` are the canonical record of patterns Toni has already proven or is actively reviewing.

When drafting a scope or SOW, scan that directory first. For each relevant synthesis doc, surface:

- the canonical implementation and source repos
- what can be reused
- what must vary by client
- what private or risky details must not be copied

When citing a pattern in a draft, reference the synthesis doc by relative path.
